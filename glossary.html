用語解説

説明

ここでは「複数の単語を繋げて一つの単語として表記する際の命名規則」を一部紹介します。

ゲームに関していうと、ケバブケースになりうる落下文字列を選ぶのに活かしてね。
※ゲーム上は、アッパーケバブケース（トレインケース）はケバブ素材に含めることとします

キャメルケース

別名：ローワーキャメルケース
＝最初の単語は小文字、以降の単語は頭が大文字で残りは小文字
例：camelCase
由来：頭を下げたラクダ
使用言語：JavaScript, Java, C# など

アッパーキャメルケース

別名：パスカルケース
＝全単語の頭文字が大文字で、あとは小文字
例：CamelCase
由来：頭を上げたラクダ
使用言語：Java, C#, Swift など

スネークケース

別名：ローワースネークケース
＝全ての単語が小文字、単語間は _ で区切る
例：snake_case
由来：蛇みたいだから
使用言語：Python, Ruby, C など

アッパースネークケース

別名：スクリーミングスネークケース、コンスタントケース
＝全ての単語が大文字、単語間は _ で区切る
例：snake_case
由来：蛇みたいだから
使用言語：Python, Ruby, C など

ケバブケース

別名：ローワーケバブケース
＝全ての単語は小文字、単語間を - で区切る
例：kebab-case
由来：串刺しみたいだから
使用言語：HTML, CSS, URL, CLI など

アッパーケバブケース

別名：トレインケース
＝各単語の最初は大文字であとは小文字、単語間を - で区切る
例：Train-Case
覚え方：頭がでた列車みたいだから
使用言語：HTML, CSS, URL, CLI など

ハンガリアン記法

＝変数名に型や用途を接頭辞として付ける
例：strText
使用言語：C, C++ など

ドットケース

＝単語を . で区切る
例：kusama.yayoi
使用言語：ファイル名、名前空間表記など
